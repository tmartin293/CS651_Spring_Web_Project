Hibernate & ORM Overview

Object-relational mapping or ORM is a technique for converting data between incompatible systems using an object-oriented programming language. In essence, ORM allows developers to work with a virtual database of objects rather than a database of another type as ORM handles the conversion between the two. For Java developers there are a number of ORM software solutions that can be used for mapping Java objects to databases. One free ORM tool is Hibernate which can map Java objects to a relational database. Hibernate allows developers to create and edit database tables with an XML configuration file.

Useful Terminology

  • Data Access Object (DAO): The Database Access Object is a pattern for separating an application from a persistent database by using an abstract API. This API allows CRUD operations that may be complex with the database language to be abstracted and hidden from the actual application.

  • Java Persistence API (JPA): The Java Persistence API is the Java standard that maps Java objects to a relational database. This data persistence API is one possible approach to implement ORM. Hibernate itself is an implementation of the JPA specification.

  • Java Database Connectivity (JDBC): The Java database Connectivity API is responsible for managing database connections, issuing queries, and handling database results.
